Start.sh Flow Map

量化交易项目启动流程全景图

这份页面严格按当前代码实现还原入口链路：从 start.sh 出发， 分流到 GUI 回测、终端批量回测、数据同步三条主路径，并补上每条路径的前置校验、 菜单行为、数据依赖、输出产物与退出规则。

主入口：start.sh 分支数：3 条主业务路径 环境门槛：必须存在 .venv/bin/python 主要产物：logs/backtest/*.html 与 data/daily/*.csv

总览路由

start.sh 是一个带主循环的菜单路由器。它先校验虚拟环境，再根据输入把流程切到 3 条子链路。 子流程正常结束会回到主菜单；若子流程主动抛出退出码 86，则整套菜单直接退出。

GUI 回测 批量回测 数据同步

入口节点：start.sh

先 cd 到项目根目录，再检查 .venv/bin/python。 如果不存在，直接提示先执行 ./bootstrap.sh 并退出，不做任何降级。

主菜单原样语义

请选择启动方式：
  1. GUI 回测
  2. 终端批量回测
  3. 拉取数据
  q. 退出

退出规则

  • 输入 q：直接结束主流程。
  • 子脚本返回 0：打印“已返回主菜单”，重新进入选择。
  • 子脚本返回 86：视为“退出整套菜单”，主脚本直接退出。
  • 其他非 0 退出码：原样向上抛出，停止主流程。

分支详情

下面按实际执行顺序展开 3 条业务链路。点击标签或上面的路由卡片即可切换查看。

GUI 回测最像“交互驾驶舱”

  • 支持策略大类、版本、股票、初始资金逐步交互选择。
  • 普通双均线家族会自动联跑全部版本，再生成家族对比 HTML。
  • 缺少股票日线 CSV 时会尝试自动调用同步脚本补数据。
  • 成功后会尝试在 macOS 自动打开报告。

批量回测强调“批处理”

  • 先选策略，再输入初始资金。
  • 直接校验所需 CSV 是否齐全，缺数据就报错并提示先执行同步。
  • 执行时关闭绘图和交易日志，更适合批量跑测试样本。
  • 对每个测试股票逐个回测，普通双均线可联跑全部版本。

数据同步承担“供血”职责

  • 支持固定代码、默认股票池、全部上证主板三种同步目标。
  • 支持东方财富、Baostock、Akshare、Tushare、自动回退链。
  • 同步日志落到 logs/sync_akshare.log。
  • 失败队列会冷却 8 秒后再重跑 1 轮，最终失败则退出码 1。

三条分支的共同约束

  • 都依赖同一个虚拟环境：.venv/bin/python。
  • 子壳脚本都会先创建 data 和 logs 目录。
  • 没有做额外兼容兜底，缺环境、缺数据、参数错误都直接失败。
  • 主菜单始终是整个项目的统一调度入口。

GUI 回测主链路

1

壳脚本准备环境

start_backtest_gui.sh 检查虚拟环境，创建 data / logs，随后执行 run_single_backtest_gui.py。

2

选择策略

从策略注册中心读取白名单策略。当前可见大类是 普通双均线 与 统计套利配对交易。其中普通双均线家族会自动联跑全部版本。

3

选择股票

支持测试样本、系统推荐前 5、以及部分策略支持的手动输入。推荐逻辑会基于本地数据先做预评估，再按年化、夏普、回撤排序。

4

输入初始资金

默认 10w，支持直接输入数值或 万/w 形式，非法输入直接报错重填。

5

校验数据并自动补数

若所需 CSV 不存在，会自动调用 sync/sync_akshare.py <code> 尝试补齐。若补数失败，GUI 流程会直接报错退出，不做静默跳过。

6

执行回测并生成 HTML

单策略输出单份报告；普通双均线家族会汇总多版本报告，额外生成一个家族总览面板。最后在 macOS 上尝试自动打开报告文件。

GUI 分支的关键产物

  • 单策略报告：logs/backtest/{report_name}-{code}.html
  • 家族对比报告：logs/backtest/simple_ma_backtest-family-{code}.html
  • 基准数据缺失时：报告继续生成，但不展示基准曲线。
  • 退出方式：用户在交互里输入 q 时，内部会抛出 86，最终让主菜单一并退出。
入口：start_backtest_gui.sh 交互：run_single_backtest_gui.py 策略注册：strategy_registry.py 回测家族：simple_ma / pair_trade

终端批量回测主链路

1

壳脚本准备环境

start_backtest.sh 同样先检查虚拟环境，再创建 data / logs 并执行 run_batch_backtest.py。

2

选择批量策略

菜单按策略大类分组显示。普通双均线在这里不是单版本选择，而是提供“联跑全部版本”的批处理入口。

3

输入初始资金

默认也是 10w。只负责解析资金，不负责兜底修正业务逻辑。

4

严格校验本地数据

批量回测不会自动同步。若任何测试股票缺少所需日线文件，直接抛出错误并要求先执行 ./sync_data.sh。

5

进入批量执行器

batch_backtest.py 会遍历策略的 TEST_CASES，逐只股票回测；执行时强制 plot=False、print_log=False，保证是批处理模式。

批量分支的业务特征

  • 适合快速对一组样本股票做统一回测，不负责自动修复前置数据问题。
  • 普通双均线会批量联跑家族内所有版本；其他策略按单一策略执行。
  • 若策略没有配置 TEST_CASES，会直接报错，不能“空跑”。
  • 和 GUI 相比，这条分支更偏“命令式批处理”，更少页面化反馈。
入口：start_backtest.sh 调度：run_batch_backtest.py 执行器：batch_backtest.py 依赖：本地日线 CSV 必须齐备

数据同步主链路

1

壳脚本准备环境

sync_data.sh 检查虚拟环境并创建目录。如果调用时自带参数，会直接透传给 Python 脚本；否则进入交互菜单。

2

选择同步目标

支持固定股票代码、默认设置数据、全部上证主板普通账户可买股票 3 种模式。

3

选择数据源

支持东方财富、Baostock、Akshare、Tushare，以及自动回退链。自动模式优先级是：东方财富直连 → Baostock → Akshare → Tushare。

4

解析目标代码与区间

Python 入口会结合命令行参数、默认股票列表、复权口径和日期范围，最终展开成一批待同步代码。

5

执行批量同步

逐个代码拉取、合并并落盘到 data/daily。若遇到失败队列，会冷却 8 秒后再重跑 1 次。

6

记录日志与退出

日志同时写终端和 logs/sync_akshare.log。最终若仍有失败代码，直接以退出码 1 结束。

同步分支的输入与输出

  • 固定代码模式：支持一个或多个股票代码，空格或逗号分隔。
  • 默认模式：从默认股票配置和基准配置中推导目标股票集。
  • 全量模式：筛选上证主板普通账户可买股票后批量同步。
  • 落盘文件：data/daily/{code}_{adjust_flag}.csv
东方财富直连 Baostock Akshare Tushare 自动回退链

环境门槛

所有主分支都硬性依赖 .venv/bin/python。缺失时不尝试系统 Python，不做兼容降级，直接要求先执行 ./bootstrap.sh。

策略白名单

当前策略注册中心只开放 普通双均线 和 统计套利配对交易 两个大类；其他策略文件存在，但不会出现在主交互菜单里。

数据依赖差异

GUI 回测会自动补缺失单票数据；终端批量回测不会自动补，只会快速失败并提示先同步。

统一退出码

86 是“退出整个菜单系统”的内部协议码。主菜单收到它后，会直接结束，而不是继续回到上一层。

页面依据的核心文件包括：start.sh、start_backtest_gui.sh、 run_single_backtest_gui.py、start_backtest.sh、 run_batch_backtest.py、batch_backtest.py、 sync_data.sh、sync/sync_akshare.py、 backtest/strategy_registry.py。
