这个是我自己想的，需要近两年年线都是涨的才适用。

平常记录指标：
1. 每次死叉，记录上一次金叉以来最高的快线值为Y 上次最高价
2. 每次金叉，记录上一次死叉以来最低的快线值为X 上次最低价

卖出逻辑:
1. 股价>=(上次最高价-买入价)*0.9+买入价后，进入待售状态
2. 待售状态下，等到连续3天下跌，或距离最近的高点下跌超3%。
3. 死叉卖出。
   1. 但是想了想，前面有个价格回升到最高价附近的卖出条件了，又回到死叉卖出，很可能只是震荡，不如等价格真正跌下来再卖出，所以先注释掉死叉卖出
4. 股价距离上次买入价跌了10%卖掉，同时清除“上次最低价”，等到下次重新记录“上次最低价”。
5. 在谨慎状态下且距离买入涨超过10%，进入待售状态
6. 当前在水上，且水上时间不足之前在水下时间的一半时，进入谨慎状态
   1. 这个要优化吗？因为只要在水面附近震荡一个来回，会很容易进入谨慎状态，不过这种时候进入谨慎状态可能更好？毕竟那么震荡。
   2. 目前谨慎状态只在发现收益够10%时才有参考作用，因为如果在水下时间久，代表主力吸收了很多筹码，不会那么容易让股价降，那么就可以多赚点。

买入逻辑:
1. 价格 <= “上次最低价” × 1.1后10个交易日内，价格 < "上次最高价" × 0.9且以下满足一项。
   1. 连续5个交易日有4个都是涨的就买入
   2. 连续5个交易日有3个都是涨的，且离决定买入当天后最低价涨幅达3%就买入
2. 价格 <= “上次最低价” × 1.1后10个交易日内，遇到金叉

## 数据

### akshare

目前用的数据是akshare，这个是开源的，不算稳定，而且它其实是一个工具，底层还是去拉各个网站的数据，特别容易被ban，所以里面 /data/cookie.txt 里面时不时要重新设置

设置方法： 进入https://quote.eastmoney.com/concept/sz000014.html，然后点击报错里的链接，找到cookie填进去。

不要在实盘里用，只能用它的历史数据验证猜想。

每次用之前可以更新，因为听说可能更新得快：
```shell
pip install akshare --upgrade -i https://pypi.tuna.tsinghua.edu.cn/simple
```
